

### ホワイトボックステスト

* ホワイトボックステストは、コードの中身を見てテストするため実装ミスを見つけやすく、カバレッジも測定できるが、本来必要な機能が実装されていない場合は見逃すことがある

#### ステートメントテスト

* コードの各ステートメント（実行可能な行）を実行することを確認するテスト技法

例）

```java
if(age > 18){
    discount = true;
}
```

テストデータ

```text
age = 20
```

↓

```java
if(age > 18){
    discount = true;
}
```

を実行できるため、ステートメントカバレッジは100%になる

ただし、

```java
if(age >= 18)
```

が正しい仕様なのに

```java
if(age > 18)
```

と誤実装されていた場合、

```text
age = 20
```

では欠陥を発見できない

→ ステートメントを実行したことは確認できるが、正しく実装されていることまでは保証できない

#### ブランチテスト

* コード内のすべてのブランチ（真・偽の判定結果）を実行することを確認するテスト技法

例）

```java
if(age >= 18){
    discount = true;
}
```

ブランチ

```text
真：age >= 18
偽：age < 18
```

テストデータ

```text
age = 20
age = 10
```

↓

真と偽の両方を実行できるため、ブランチカバレッジは100%になる

ただし、

```java
if(A){
    処理1;
}

if(B){
    処理2;
}
```

の場合、

```text
TC1：A=true、B=false
TC2：A=false、B=true
```

でブランチカバレッジ100%になるが、

```text
A=true、B=true
```

の時だけ発生する欠陥は検出できない可能性がある

→ ブランチを実行したことは確認できるが、すべてのパス（組み合わせ）を確認したことにはならない

#### カバレッジの関係

* ブランチカバレッジ100%

  * ステートメントカバレッジ100%

* ステートメントカバレッジ100%

  * ブランチカバレッジ100%ではない
