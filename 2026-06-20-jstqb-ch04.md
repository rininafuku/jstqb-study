

## ホワイトボックステスト

* ホワイトボックステストは、コードの中身を見てテストするため実装ミスを見つけやすく、カバレッジも測定できるが、本来必要な機能が実装されていない場合は見逃すことがある

#### ステートメントテスト

* コードの各ステートメント（実行可能な行）を実行することを確認するテスト技法

例）

```java
if(age > 18){
    discount = true;
}
```

テストデータ

```text
age = 20
```

↓

```java
if(age > 18){
    discount = true;
}
```

を実行できるため、ステートメントカバレッジは100%になる

ただし、

```java
if(age >= 18)
```

が正しい仕様なのに

```java
if(age > 18)
```

と誤実装されていた場合、

```text
age = 20
```

では欠陥を発見できない

→ ステートメントを実行したことは確認できるが、正しく実装されていることまでは保証できない

#### ブランチテスト

* コード内のすべてのブランチ（真・偽の判定結果）を実行することを確認するテスト技法

例）

```java
if(age >= 18){
    discount = true;
}
```

ブランチ

```text
真：age >= 18
偽：age < 18
```

テストデータ

```text
age = 20
age = 10
```

↓

真と偽の両方を実行できるため、ブランチカバレッジは100%になる

ただし、

```java
if(A){
    処理1;
}

if(B){
    処理2;
}
```

の場合、

```text
TC1：A=true、B=false
TC2：A=false、B=true
```

でブランチカバレッジ100%になるが、

```text
A=true、B=true
```

の時だけ発生する欠陥は検出できない可能性がある

→ ブランチを実行したことは確認できるが、すべてのパス（組み合わせ）を確認したことにはならない

#### カバレッジの関係

* ブランチカバレッジ100%

  * ステートメントカバレッジ100%

* ステートメントカバレッジ100%

  * ブランチカバレッジ100%ではない



## 経験ベースのテスト技法

経験ベースのテスト技法は、テスト担当者の経験や知識を利用してテストケースを作成する技法である

##### エラー推測法

* アプリケーションの過去の動作状況、開発担当者が起こしやすいエラー、類似アプリケーションで発生した故障などの経験や知識を基に、欠陥がありそうな箇所を予測してテストする

###### メリット

* 経験を活用して効率よく欠陥を発見できる

###### デメリット

* テスト担当者の経験や知識に大きく依存する

###### フォールト攻撃

* エラー推測を体系的に行う方法であり、想定されるエラー、欠陥、故障のリストを基にテストケースを設計する

##### 探索的テスト

* テスト対象について学習しながら、テストケースを同時に設計、実行、評価し、まだテストしていない領域を探索する

###### セッションベースドテスト

* 探索的テストを体系的に行う方法であり、あらかじめ決めた時間枠（テストセッション）の中で、テストチャーター（テスト目的や重点的に確認する内容を記載したもの）に従って実施する。テスト担当者は実施内容や発見事項を記録し、終了後にデブリーフィング（振り返り）で結果を共有する

* 探索的テストは、仕様がほとんどない場合や不十分な場合、テストのスケジュールに余裕がない場合に有効であり、他のテスト技法（同値分割法など）と併用することもできる。また、経験、ドメイン知識、分析力、好奇心、創造性を持つテスト担当者ほど効果が高い

##### チェックリストベースドテスト

* 経験や過去の障害を基に作成したチェックリストを利用してテストする技法

* チェックリストは、経験、ユーザーにとって重要なこと、ソフトウェアが不合格になる理由や仕組み（過去の障害や発生しやすい欠陥など）を基に作成する

###### メリット

* 詳細なテストケースがなくてもテストのガイドラインとなり、抜け漏れ防止や一定の品質確保に役立つ

###### デメリット

* チェックリストが大まかだとテスト担当者によって実施内容にばらつきが生じ、再現性が低くなる場合がある

* チェックリストは欠陥分析を基に定期的に更新する必要がある

###### 更新が必要な理由

* 開発担当者は同じエラーを避けることを学ぶため、既存のチェック項目の効果が薄れることがある。また、新たに発見した重要度の高い欠陥を反映する必要があるため
